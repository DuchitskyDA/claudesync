# claudesync v0.3 — Plugin Manager + Plugin Presets

**Дата:** 2026-05-07
**Статус:** Draft
**Goal:** Дать пользователю GUI для установки/удаления Claude Code плагинов одной кнопкой. Каталог известных плагинов хостится в отдельном GitHub repo, обновляется без релиза приложения. Плагины с required env (например Context7 API key) предлагают модалку с инструкцией. Пресеты — именованные группы плагинов, активируются вместе.

## Контракт с Claude Code

Claude Code читает `<RULES_TARGET>/settings.json` (по умолчанию `~/.claude/settings.json`). Плагины активируются через 3 поля:

```jsonc
{
  "env": {
    "CONTEXT7_API_KEY": "ctx7sk-..." // env vars нужные плагинам
  },
  "enabledPlugins": {
    "superpowers@claude-plugins-official": true,        // built-in marketplace
    "frontend-design@claude-code-plugins": true,        // нужно добавить marketplace в extraKnownMarketplaces
    "template-bridge@template-bridge-marketplace": true
  },
  "extraKnownMarketplaces": {
    "claude-code-plugins": {
      "source": { "source": "github", "repo": "anthropics/claude-code" }
    },
    "template-bridge-marketplace": {
      "source": { "source": "github", "repo": "maslennikov-ig/template-bridge" }
    }
  }
}
```

При следующем старте Claude Code:
1. Читает settings.json
2. Для каждого `extraKnownMarketplaces` клонирует marketplace в `~/.claude/plugins/cache/<marketplace>/`
3. Для каждого активного плагина устанавливает его в `~/.claude/plugins/cache/<marketplace>/<plugin>/<version>/`
4. Активирует skills/commands/MCP servers плагина

**Никаких CLI команд `/plugin install ...` не требуется** — Claude Code сам всё делает по конфигу.

## Каталог плагинов

Хранится в отдельном репо `DuchitskyDA/claudesync-plugins` (или внутри основного, если удобнее) — JSON, обновляется через PR без релиза приложения.

**Schema:**

```ts
type PluginCatalog = {
  version: 1
  plugins: PluginEntry[]
  presets: PresetEntry[]
}

type PluginEntry = {
  id: string                              // полный идентификатор для enabledPlugins, e.g. "frontend-design@claude-code-plugins"
  name: string                            // человекочитаемое имя
  description: string                     // 1-2 предложения
  author?: string
  homepage?: string
  tags?: string[]                         // ["workflow", "skills", "agents", "docs", "ui"]
  marketplace?: {                         // null если built-in (claude-plugins-official)
    id: string                            // ключ для extraKnownMarketplaces
    source: { source: 'github'; repo: string }
  }
  requiresEnv?: EnvRequirement[]          // env vars нужные плагину
  recommendedFor?: string[]               // ["frontend", "backend", "writing"]
}

type EnvRequirement = {
  name: string                            // имя env var (CONTEXT7_API_KEY)
  label: string                           // отображаемое имя ("Context7 API Key")
  instructions: string                    // markdown — как получить
  placeholder?: string                    // для input field
  docsUrl?: string                        // ссылка для кнопки "Open docs"
  optional?: boolean                      // если true — можно skip plugin вместо ввода
}

type PresetEntry = {
  id: string
  name: string                            // "Frontend Dev Stack"
  description: string
  pluginIds: string[]                     // ссылки на plugins[].id
  icon?: string
}
```

### Preset apply flow

1. Клик [Apply] на пресет
2. Все его plugin checkboxes отмечаются
3. Если у каких-то требуется env — последовательно открываются env-модалки
4. После всех env собрана картина → выполняется apply changes
5. Toast: "Activated 3 plugins. Restart Claude Code to apply."

## Tool detection

Plugin manager работает только если `<rulesTarget>/settings.json` существует и валидный JSON, ИЛИ если файла нет — приложение предложит инициализировать пустой `{}`.

Если rulesTarget явно не Claude (`~/.cursor`, `~/.aider`) — Plugins tab disabled с tooltip "Plugin manager available only for Claude Code rulesTarget".

## Безопасность settings.json

Атомарная запись через `<file>.tmp` + rename — уже реализовано в `config.ts`. Расширим логику:
- **Read** существующий settings.json (или пустой `{}` если нет)
- **Merge** изменения в `enabledPlugins`, `extraKnownMarketplaces`, `env`
- **Не трогать** другие поля (`permissions`, `hooks`, `effortLevel` и пр.)
- **Atomic write**

Если settings.json существует, но не валидный JSON — показать ошибку и предложить открыть файл руками для починки.

## Restart Claude Code prompt

Изменения в settings.json подхватываются Claude Code только при следующем запуске. После Apply показываем toast/banner:

```
Plugins updated. Restart Claude Code to apply.
[ Got it ]
```

## API расширение

Новые IPC channels:
- `get-plugin-catalog`: `() => Promise<PluginCatalog>` (с кешем)
- `get-installed-plugins`: `() => Promise<{ enabledIds: string[]; envSet: string[] }>` (читает settings.json)
- `apply-plugin-changes`: `(changes: { enable: PluginEntry[]; disable: string[]; envValues: Record<string, string> }) => Promise<{ ok: boolean; error?: string }>`
- `validate-claude-target`: `() => Promise<{ ok: boolean; reason?: string }>` — проверяет что rulesTarget похож на Claude Code dir

## Out of scope для v0.3

- **Skill picker** — выбирать какие конкретные skills из плагина enable. Плагины активируются целиком.
- **Plugin updates** — Claude Code сам обновляет cached marketplaces при `/plugin marketplace update`. Полагаемся на это.
- **Удаление плагина из cache** — пользователь делает руками, мы только меняем `enabledPlugins`.
- **Marketplace ratings/popularity** — позже (v0.5+, marketplace становится social).
- **Несколько профилей** — отдельная фича (v0.4 profiles).
