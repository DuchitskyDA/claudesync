# claudesync v0.4 — Bidirectional Sync + Initialize Wizard

**Дата:** 2026-05-08
**Статус:** Approved (pending user review of this document)
**Goal:** Превратить Sync из односторонней (Pull) в двустороннюю операцию. Добавить Initialize wizard для создания GitHub repo из локального `~/.claude/` одной кнопкой. Cross-OS (Mac/Win/Linux). v0.5 добавит three-way merge UI; в v0.4 conflict handling = «resolve manually + Show in Folder».

## Контекст

После v0.3:
- Sync делает только Pull: `git clone || git pull` + run `install.sh`/`install.ps1` с `RULES_TARGET` env.
- Установка через симлинки (Mac/Linux/Win-DevMode) или копии (Win без DevMode).
- Plugin manager пишет в `<rulesTarget>/settings.json` (не в repo).
- Convention репозитория: `<repo>/global/{CLAUDE.md, settings.json, commands/, skills/, projects/<encoded>/memory/}` + `install.sh`/`install.ps1` в корне.

v0.4 добавляет:
- **Initialize Wizard** — создать новый GitHub repo из текущего `~/.claude/` одной операцией.
- **Push** — экспортировать локальные изменения в существующий repo.
- **GitHub OAuth Device Flow** — auth для create/push без `gh` CLI зависимости.

## Convention contract

claudesync v0.4 заточен под **hardcoded convention**:

```
<repo>/
├── global/
│   ├── CLAUDE.md                          (file)
│   ├── settings.json                      (file)
│   ├── commands/*.md                      (flat files)
│   ├── skills/<name>/                     (directories)
│   └── projects/<encoded-path>/memory/    (per-machine memory dirs)
├── install.sh                             (RULES_TARGET-aware)
├── install.ps1                            (RULES_TARGET-aware)
├── README.md
├── LICENSE
└── .gitignore
```

`<encoded-path>` = path to user's project with `/` and `\` replaced by `-`. Например:
- macOS: `/Users/dan/work/foo` → `-Users-dan-work-foo`
- Windows: `C:\Users\dan\work\foo` → `C--Users-dan-work-foo`

Memory папки **не пересекаются между ОС** — это by design.

Импорт «чужих» репо с другой структурой Push не поддерживает (только Pull, как раньше).

## OAuth Device Flow

**Scopes:** `repo` (create repos + push), `read:user` (display login in UI).

**Flow:**

1. User clicks "Sign in with GitHub" in InitWizard step 1
2. main calls POST `https://github.com/login/device/code` with `{ client_id, scope }`
3. Response: `{ device_code, user_code, verification_uri, interval, expires_in }`
4. main stores `device_code` in module memory (not persisted)
5. main returns `{ userCode, verificationUri, expiresIn, interval }` to renderer
6. DeviceFlowModal shows code + "Open browser" button
7. User opens browser, enters code, authorizes
8. main polls every `interval` sec via `pollDeviceFlow`:
   - POST `https://github.com/login/oauth/access_token` with `{ client_id, device_code, grant_type: 'urn:ietf:params:oauth:grant-type:device_code' }`
   - 400 `authorization_pending` → continue waiting
   - 400 `slow_down` → increase interval by 5s
   - 400 `expired_token` → fail
   - 200 `{ access_token, scope }` → save token via safeStorage, return ok

**Token storage (`src/main/safe-storage.ts`):**

```ts
import { safeStorage } from 'electron'
import { readFileSync, writeFileSync, unlinkSync, existsSync } from 'node:fs'
import { join } from 'node:path'

export function saveToken(userDataDir: string, token: string): void {
  const encrypted = safeStorage.encryptString(token)
  writeFileSync(join(userDataDir, 'github-credentials.bin'), encrypted)
}

export function loadToken(userDataDir: string): string | null {
  const path = join(userDataDir, 'github-credentials.bin')
  if (!existsSync(path)) return null
  try {
    return safeStorage.decryptString(readFileSync(path))
  } catch {
    return null
  }
}

export function deleteToken(userDataDir: string): void {
  const path = join(userDataDir, 'github-credentials.bin')
  if (existsSync(path)) unlinkSync(path)
}
```

`safeStorage` использует:
- macOS: Keychain
- Windows: DPAPI (binds to Windows user account)
- Linux: libsecret / gnome-keyring

**Token usage for git push:**

```ts
spawn('git', ['push', 'origin', 'main'], {
  cwd: repoPath,
  env: { ...process.env, GITHUB_TOKEN: token, GIT_ASKPASS: 'echo' },
})
```

Git с `GIT_ASKPASS=echo` + remote URL `https://github.com/...` подхватит `GITHUB_TOKEN` через `credential.helper` (or fallback).

Альтернатива (более прямая): инжектить токен в URL через temp remote setting:
```
git -c http.https://github.com/.extraheader="Authorization: Bearer <token>" push ...
```
Этот вариант не записывает токен в `.git/config`, передаётся только на одно invocation.

**Используем второй вариант** для max-cleanliness.

## GitHub API (`src/main/github-api.ts`)

```ts
const API_BASE = 'https://api.github.com'

async function ghApi(token: string, path: string, init?: RequestInit) {
  const r = await fetch(`${API_BASE}${path}`, {
    ...init,
    headers: {
      Authorization: `Bearer ${token}`,
      Accept: 'application/vnd.github+json',
      'X-GitHub-Api-Version': '2022-11-28',
      ...(init?.headers ?? {}),
    },
  })
  if (!r.ok) throw new Error(`GitHub API ${r.status}: ${await r.text()}`)
  return r.json()
}

export async function getUser(token: string): Promise<{ login: string; ... }> {
  return ghApi(token, '/user')
}

export async function listOrgs(token: string): Promise<{ login: string }[]> {
  return ghApi(token, '/user/orgs')
}

export async function createRepo(
  token: string,
  opts: { owner: string; name: string; isPrivate: boolean; description?: string },
): Promise<{ clone_url: string; html_url: string; full_name: string }> {
  // user vs org endpoint
  const user = await getUser(token)
  const isUserRepo = opts.owner === user.login
  const path = isUserRepo ? '/user/repos' : `/orgs/${opts.owner}/repos`
  return ghApi(token, path, {
    method: 'POST',
    body: JSON.stringify({
      name: opts.name,
      private: opts.isPrivate,
      description: opts.description,
      auto_init: false,
    }),
  })
}
```

## Initialize Wizard

### UI — 4 шага

**Step 1 — Sign in to GitHub:**
- Если `getAuthState().authenticated === true` → автопереход к Step 2
- Иначе кнопка "Sign in with GitHub" → DeviceFlowModal
- После успеха — отображение `@login` + кнопка "Continue"

**Step 2 — Repo settings:**
- Repo name (text input, default `claudesync-config`, regex validation)
- Owner dropdown (loaded from `listOwners`: personal + orgs)
- Visibility radio (Public / Private, default Private)
- Description (optional, single line)
- "Continue" disabled until name valid + owner selected

**Step 3 — Preview:**
- Loaded from `scanLocalConfig()`
- Two columns:
  - Files to include (✓): `global/CLAUDE.md`, `global/settings.json`, `global/commands/*` (count), `global/skills/*` (count), `global/projects/*/memory` (count)
  - Excluded (✗): `sessions/`, `cache/`, `*.jsonl`, `.credentials.json`, etc
- Notice: «env block in settings.json WILL BE STRIPPED — re-add API keys via Plugin manager after pushing»
- Кнопки [Back] [Create & Push]

**Step 4 — Progress:**
- Streaming step list (как Sync tab):
  - ⟳ Creating repo on GitHub
  - ⟳ Cloning empty repo
  - ⟳ Generating global/ structure
  - ⟳ Initial commit
  - ⟳ Pushing
  - ✓ Done!
- При ошибке: красная строка с message + кнопка "Try again" (возвращает на step 2 с заполненными полями)
- При успехе: ссылка "Open repo" + "Close"

### Под капотом — `init-wizard.ts`

```ts
export async function initRepo(opts: {
  ownerLogin: string
  name: string
  isPrivate: boolean
  description?: string
  rulesTarget: string
  userDataDir: string
  emit: (line: LogLine) => void
  emitStep: (e: { step: string; status: StepStatus; message?: string }) => void
}): Promise<RunResult> {
  const token = loadToken(opts.userDataDir)
  if (!token) return fail('Not signed in')

  // 1. Create repo via API
  emitStep({ step: 'create-repo', status: 'running' })
  let cloneUrl: string
  try {
    const repo = await createRepo(token, opts)
    cloneUrl = repo.clone_url
    emitStep({ step: 'create-repo', status: 'done' })
  } catch (e) {
    emitStep({ step: 'create-repo', status: 'failed', message: e.message })
    return fail(e.message)
  }

  // 2. Compute local managed path
  const localPath = defaultManagedRepoPath(cloneUrl, opts.userDataDir)
  mkdirSync(dirname(localPath), { recursive: true })

  // 3. Clone (empty repo)
  emitStep({ step: 'clone', status: 'running' })
  const cloneResult = await runCommand('git', ['clone', cloneUrl, localPath], {
    cwd: dirname(localPath),
    env: tokenEnv(token),
    onLine: opts.emit,
  })
  if (cloneResult.exitCode !== 0) {
    emitStep({ step: 'clone', status: 'failed', message: 'clone failed' })
    return fail(`git clone failed: exit ${cloneResult.exitCode}`)
  }
  emitStep({ step: 'clone', status: 'done' })

  // 4. Generate structure from rulesTarget
  emitStep({ step: 'generate', status: 'running' })
  await generateGlobalStructure(opts.rulesTarget, localPath)
  await dropTemplates(localPath, { name: opts.name, owner: opts.ownerLogin })
  emitStep({ step: 'generate', status: 'done' })

  // 5. Initial commit
  emitStep({ step: 'commit', status: 'running' })
  await runCommand('git', ['-C', localPath, 'add', '.'], { ... })
  const commit = await runCommand('git', ['-C', localPath, 'commit', '-m', 'initial commit from claudesync'], { ... })
  if (commit.exitCode !== 0) {
    emitStep({ step: 'commit', status: 'failed' })
    return fail('commit failed')
  }
  emitStep({ step: 'commit', status: 'done' })

  // 6. Push
  emitStep({ step: 'push', status: 'running' })
  const push = await runCommand('git', ['-C', localPath, 'push', '-u', 'origin', 'main'], {
    env: tokenEnv(token),
    onLine: opts.emit,
  })
  if (push.exitCode !== 0) {
    emitStep({ step: 'push', status: 'failed' })
    return fail(`push failed: exit ${push.exitCode}`)
  }
  emitStep({ step: 'push', status: 'done' })

  // 7. Update config
  writeConfig({ repoUrl: cloneUrl, repoPath: localPath, rulesTarget: opts.rulesTarget })

  return { ok: true, exitCode: 0 }
}
```

### `generateGlobalStructure(rulesTarget, repoPath)` — копирование

```ts
const GLOBAL_DIR = join(repoPath, 'global')
mkdirSync(GLOBAL_DIR, { recursive: true })

// Files
copyIfExists(join(rulesTarget, 'CLAUDE.md'), join(GLOBAL_DIR, 'CLAUDE.md'))

// settings.json — strip env
const settings = readClaudeSettings(join(rulesTarget, 'settings.json'))
const sanitized = { ...settings, env: undefined }
delete sanitized.env
writeFileSync(join(GLOBAL_DIR, 'settings.json'), JSON.stringify(sanitized, null, 2))

// Dirs (commands, skills) — full mirror
copyDirIfExists(join(rulesTarget, 'commands'), join(GLOBAL_DIR, 'commands'))
copyDirIfExists(join(rulesTarget, 'skills'), join(GLOBAL_DIR, 'skills'))

// projects/*/memory — selective
const projectsSrc = join(rulesTarget, 'projects')
const projectsDst = join(GLOBAL_DIR, 'projects')
if (existsSync(projectsSrc)) {
  for (const dir of readdirSync(projectsSrc)) {
    const memorySrc = join(projectsSrc, dir, 'memory')
    if (existsSync(memorySrc)) {
      copyDirSync(memorySrc, join(projectsDst, dir, 'memory'))
    }
  }
}
```

## Push Flow

### `push.ts`

```ts
export async function runPush(opts: {
  configPath: string
  userDataDir: string
  includeSecrets: boolean
  commitMessage: string
  emit: (line: LogLine) => void
  emitStep: (e: { step: PushStep; status: StepStatus; message?: string }) => void
}): Promise<RunResult> {
  const cfg = readConfig(opts.configPath)
  if (!cfg.repoUrl || !cfg.repoPath || !cfg.rulesTarget) return fail('Sync not configured')

  const token = loadToken(opts.userDataDir)
  if (!token) return fail('Not signed in to GitHub')

  return withRunLock(async () => {
    // 1. Detect install mode (symlink vs copy)
    emitStep({ step: 'export', status: 'running' })
    const probe = join(cfg.repoPath, 'global', 'CLAUDE.md')
    const needsExport = !existsSync(probe) || !lstatSync(probe).isSymbolicLink()

    if (needsExport) {
      await exportRulesToRepo(cfg.rulesTarget, cfg.repoPath)
    }
    emitStep({ step: 'export', status: 'done' })

    // 2. Strip secrets unless includeSecrets
    if (!opts.includeSecrets) {
      stripSecretsInRepo(cfg.repoPath)
    }

    // 3. Check if anything to push
    const status = await runCommand('git', ['-C', cfg.repoPath, 'status', '--porcelain'], { ... })
    if (statusOutput.trim() === '') {
      return { ok: true, exitCode: 0, error: 'Nothing to push' }
    }

    // 4. Pull-rebase
    emitStep({ step: 'pull', status: 'running' })
    const rebase = await runCommand('git', ['-C', cfg.repoPath, 'pull', '--rebase', '--autostash'], {
      env: tokenEnv(token),
      onLine: emit,
    })
    if (rebase.exitCode !== 0) {
      await runCommand('git', ['-C', cfg.repoPath, 'rebase', '--abort'], { ... }).catch(() => {})
      emitStep({ step: 'pull', status: 'failed' })
      return fail(`Conflict during rebase. Resolve manually in ${cfg.repoPath} then push again.`)
    }
    emitStep({ step: 'pull', status: 'done' })

    // 5. Stage + commit
    emitStep({ step: 'commit', status: 'running' })
    await runCommand('git', ['-C', cfg.repoPath, 'add', '-A'], { ... })
    const commit = await runCommand(
      'git',
      ['-C', cfg.repoPath, 'commit', '-m', opts.commitMessage],
      { ... },
    )
    if (commit.exitCode !== 0) {
      emitStep({ step: 'commit', status: 'failed' })
      return fail('Commit failed')
    }
    emitStep({ step: 'commit', status: 'done' })

    // 6. Push
    emitStep({ step: 'push', status: 'running' })
    const push = await runCommand('git', ['-C', cfg.repoPath, 'push', 'origin', 'main'], {
      env: tokenEnv(token),
      onLine: emit,
    })
    if (push.exitCode !== 0) {
      emitStep({ step: 'push', status: 'failed' })
      return fail(`Push failed: exit ${push.exitCode}`)
    }
    emitStep({ step: 'push', status: 'done' })

    return { ok: true, exitCode: 0 }
  }).catch((e: Error) => fail(e.message))
}
```

### Export logic — hardcoded mirror

```ts
const SYNC_MAP = [
  { from: 'CLAUDE.md', to: 'global/CLAUDE.md', kind: 'file' },
  { from: 'settings.json', to: 'global/settings.json', kind: 'file' },
  { from: 'commands', to: 'global/commands', kind: 'dir' },
  { from: 'skills', to: 'global/skills', kind: 'dir' },
  { from: 'projects', to: 'global/projects', kind: 'projects-memory-only' },
]

function exportRulesToRepo(rulesTarget: string, repoPath: string): void {
  for (const m of SYNC_MAP) {
    const src = join(rulesTarget, m.from)
    const dst = join(repoPath, m.to)
    if (m.kind === 'file' && existsSync(src)) {
      copyFileSync(src, dst)
    } else if (m.kind === 'dir' && existsSync(src)) {
      mirrorDir(src, dst)
    } else if (m.kind === 'projects-memory-only' && existsSync(src)) {
      // Only sync projects/*/memory, not other subfolders (sessions, *.jsonl)
      mirrorProjectsMemory(src, dst)
    }
  }
}
```

`mirrorDir(src, dst)`:
- Walk `src`, copy files to `dst` (overwrite)
- Walk `dst`, delete files/dirs that don't exist in `src` (true mirror — repo == source of truth after export)

`mirrorProjectsMemory`:
- Iterate dirs in `src/projects/`
- For each `<encoded>/memory/` if exists in src — mirror to `dst/<encoded>/memory/`
- Don't touch other contents in `dst/<encoded>/` (e.g. `sessions/` files which never go to repo)

### Secrets stripping — `stripSecretsInRepo`

```ts
function stripSecretsInRepo(repoPath: string): void {
  const settingsPath = join(repoPath, 'global', 'settings.json')
  if (!existsSync(settingsPath)) return
  let s
  try {
    s = JSON.parse(readFileSync(settingsPath, 'utf8'))
  } catch {
    throw new Error('Invalid settings.json — fix it before push')
  }
  if (s.env) {
    delete s.env
    writeFileSync(settingsPath, JSON.stringify(s, null, 2), 'utf8')
  }
}
```

## Error Handling

| Scenario | Behavior |
|---|---|
| Token expired (401 on git push or API) | UI prompt re-auth via DeviceFlowModal |
| Device flow timeout (15 min) | Modal shows "Authorization expired, try again" |
| Repo with same name exists (422) | Step 2 inline error: "Repo already exists" |
| No permission to org (403) | Step 2 inline error |
| Empty `~/.claude/` config | Step 3 warning: "Config looks empty, continue anyway?" |
| `git status` clean | Push: toast "Nothing to push" |
| Rebase conflict | Auto-abort, fail with `Resolve manually in <path>` + Show in Folder |
| Network failure (clone/push) | Step status failed with exit code |
| Invalid JSON in settings.json | Push aborts with message + Show in Folder |
| User cancels DeviceFlowModal | Polling stops, no token saved |
| Token revoked mid-operation | 401, prompt re-auth |
| Plugin manager active during Push | `withRunLock` handles serialization |

### Manual E2E

1. **Cold start, no repo**: install claudesync → InitWizard → real repo on GitHub → push initial commit
2. **Pull on second machine**: configure URL → Pull & Install → verify symlinks
3. **Local edit + Push**: edit CLAUDE.md → Push with commit msg → verify GitHub commit, env stripped
4. **Push with includeSecrets toggle**: enable, push → verify env in repo, then revert
5. **Conflict scenario**: two machines edit same line → second push → conflict → abort + Show in Folder
6. **Token expiry**: revoke token in GitHub settings → push → 401 → re-auth flow
7. **Cross-OS**: Mac push → Win pull → settings.json/CLAUDE.md identical, memory dirs separate
8. **Plugin manager + Push**: install plugin → settings.json changes → push (no secrets) → enabledPlugins in repo
