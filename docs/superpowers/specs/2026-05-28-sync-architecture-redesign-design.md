## 1. Боли (почему вообще трогаем)

Пользователь несколько раз сам затирал важные данные при тестировании синка. Это **не россыпь багов**, а одно структурное свойство текущей модели:

> **Синк выводит «удаление» из «отсутствия».**

Текущая модель — *snapshot mirror*: на каждый push заново энумерятся живые папки (`~/.claude`, проектные `.cursor`/`.claude`), и diff против репы решает, что изменилось. Если файл не попал в снимок — движок не отличает «юзер удалил» от «не смог прочитать» и вырезает его из репы.

### Конкретные находки аудита (доказательная база для под-проекта №1)

**F1 — CRITICAL: «не смог прочитать» = «удалено».**
Любой файл, выпавший из энумерации, помечается `deleted` и вырезается из репы на push:
- битый JSON в `settings.json` → `catch { return }` в [`source-enum.ts:68`](../../../src/main/sync/engine/source-enum.ts) (и `:110` для проектного)
- ошибка чтения (lock/EPERM/EBUSY — частое на Windows, когда Claude Code пишет файл) → [`source-enum.ts:66,108`](../../../src/main/sync/engine/source-enum.ts)
- файл > 5 МБ → [`source-enum.ts:64,106`](../../../src/main/sync/engine/source-enum.ts) (memory-файлы растут)
- битый симлинк → пропуск в `walk`
Цепочка: пропуск → компаратор видит «есть в HEAD, нет в source» → `status: 'deleted'` [`comparator.ts:64-65`](../../../src/main/sync/engine/comparator.ts) → push `updateIndexRemove` [`index-builder.ts:24-25`](../../../src/main/sync/engine/index-builder.ts).
Guard `push.error.invalidJson` существует в локалях, но **не используется в коде** (мёртвое наследие удалённого legacy-push) — pre-push валидации JSON нет.

**F1b — CATASTROPHIC подслучай:** если `claude.enabled=true`, но путь временно недоступен, [`source-enum.ts:56`](../../../src/main/sync/engine/source-enum.ts) `if (!existsSync(claudePath)) return []` вернёт пустой список → весь `claude/` помечается к удалению → push стирает всё дерево. Floor-предохранителя нет.

**F1-surfacing:** push-preview показывает плоский список путей без пометки «удаление» — `preview-push-status` схлопывает diff в `repoPath`, теряя `status` [`ipc.ts:450-452`](../../../src/main/ipc.ts). Пользователь не видит, что файл удаляется, а не меняется.

**F2 — MEDIUM:** кнопка Install (`install.sh`/`install.ps1`) не разворачивает проектные `projects/<name>/.claude/` — скрипты хардкодят список (global + `projects/*/memory`), [`install.sh.template:35-68`](../../../src/main/templates/install.sh.template). Pull-флоу (`executePullApply`) их разворачивает, Install — нет. Рассинхрон двух reverse-mirror путей.

**F3 — MEDIUM:** `install.sh` `commands/*.md` нерекурсивный — вложенные `commands/sub/foo.md` пушатся (walk рекурсивный), но симлинками не разворачиваются.

**F4 — MEDIUM:** Discard удаляет локально-новые ('added') файлы [`engine.ts:427-429`](../../../src/main/sync/engine/engine.ts) `applyToSource(null)`. За confirm-диалогом, но формулировка не предупреждает, что новые memory/skills/commands будут стёрты.

**F5 — LOW/by-design:** pull заменяет allow-list ключи `settings.json` [`pull-apply.ts:34-37`](../../../src/main/sync/engine/pull-apply.ts): ключ из whitelist, которого нет в версии репы, удаляется локально. Защищено diverged-guard'ом.

**F6 — LOW:** includeSecrets пишет API-ключи в репу (предупреждение в UI есть).

### Что УЖЕ безопасно (сохранить при редизайне)
- Удаления при pull — **opt-in** ([`PullModal.tsx:23,45`](../../../src/renderer/components/PullModal.tsx), [`engine.ts:394-397`](../../../src/main/sync/engine/engine.ts)).
- Diverged-state **блокирует pull** ([`engine.ts:154`](../../../src/main/sync/engine/engine.ts), `computePullPreview:268`).
- Cursor reverse-mirror **аддитивный** ([`cursor-install.ts:7-24`](../../../src/main/sync/cursor-install.ts) — фикс прошлой потери данных в Discard).
- Claude install **бэкапит реальные файлы** перед симлинком ([`install.sh.template:16-25`](../../../src/main/templates/install.sh.template), [`install.ps1.template:21-33`](../../../src/main/templates/install.ps1.template)).
- `git clean` scoped только в `claude/`+`cursor/` ([`git-ops.ts:179-189`](../../../src/main/sync/engine/git-ops.ts)).
- Toggle-off / незарегистрированный проект симметрично исключаются из push-HEAD-фильтра И pull ([`engine.ts:60-80, 315-335`](../../../src/main/sync/engine/engine.ts)) — toggle off ≠ удаление.
- Push откатывает ref при сетевой ошибке ([`engine.ts:249-251`](../../../src/main/sync/engine/engine.ts)).

## 2. Цели и видение

1. **Бесшовный личный cross-device синк** между 2 устройствами: часто добавляются правила, шарится контекст чатов/фич/memory. (Near-term приоритет.)
2. **AI-конфиги и контекст** — глобальные И по-проектные — для Claude и Cursor (позже возможно GPT/Codex).
3. **Шарить плагины, в будущем MCP** — «установка в один клик».
4. **Внутрикомандный синк-менеджер** (продуктовое видение): команда устанавливает плагины/MCP и получает контекст/правила «в пару кликов».
5. **Тонкий конфиг «что синхронить под капотом».** Ключевой сценарий: засинкать конкретную папку/файл (напр. контекст EPIC'а с транскрибациями встреч и комментами трекера) на всю команду, отключив всё остальное; но при желании — и глобальные правила, плагины, MCP, проектные правила.
6. **Главное требование к качеству:** синк проходит очень бережно, ничего не пропадает.

## 3. Решения брейншторма (зафиксированы)

| # | Развилка | Решение | Почему |
|---|---|---|---|
| 1 | Топология | **Раздельные источники** (мульти-репо). Личная репа сейчас, командные — слой поверх. | Личная память не должна лежать в одном git-history с командным; разная аудитория/права. |
| 2 | Фундамент | **B — манифест-зеркало + предохранители.** (НЕ симлинки.) | Единая модель покрывает secrets, проектные папки, произвольные командные папки и мульти-источник. Эволюция работающего кода. Симлинки ломаются на settings.json (нужен merge), проектных/командных папках, rewrite-by-rename. |
| 3 | Где манифест | **Слоями:** репа предлагает (offered set), устройство активирует + добавляет device-local. | Даёт «лид задал EPIC-скоуп один раз, команда наследует, но каждый может отключить у себя». |
| 4 | Плагины/MCP | **Заложить в схему сейчас, исполнитель позже.** | desired-state, не файлы. YAGNI на исполнении, без переделок схемы. |

**Принцип-фундамент (над всеми решениями):** удаление — всегда явное намеренное событие; его нельзя выводить из отсутствия, ошибки чтения или фильтра. Источник правды дополняется и сливается, не разрушается без явного намерения.

## 4. Целевая архитектура

### 4.1 Ядро модели и инварианты безопасности — «tracked set + явные намерения»

Репа зеркалит только то, что описано в манифесте; движок никогда не выводит удаление из отсутствия.

1. **Членство явное (манифест), не «обнаруженное».** Путь синкается ⇔ матчит запись манифеста. Всё вне манифеста невидимо для синка.
2. **Удаление — явное событие.** Пропавший tracked-файл показывается как *кандидат* на удаление (явный чекбокс-подтверждение), не вырезается сам. Перестать синкать = убрать из манифеста.
3. **Ошибка чтения ≠ удаление.** Не прочитался tracked-файл (lock/битый JSON/>лимита) → версия в репе сохраняется + видимый warning. ← прямой фикс F1.
4. **Floor на массовые изменения.** Если push удаляет большую долю tracked-файлов источника → отказ + явное подтверждение. ← фикс F1b.
5. **Атомарная, аддитивная запись на apply.** temp+rename, никогда частично; reverse-mirror аддитивный (не удаляет local-only) кроме явного opt-in.
6. **Симметрия push и pull.** Одни правила членства/удаления в обе стороны → выключение источника/категории на одном устройстве не удаляет контент из репы или у других.

**Итог:** данные покидают репу только если (а) убраны из манифеста, или (б) явно отмечены к удалению в preview.

### 4.2 Манифест, источники, скоупы

**Источник** = git-репа. Манифест лежит в репе (`.claudesync/manifest.json`), описывает что источник *предлагает*.

**Запись манифеста — два типа:**
- **Файловая:** поверхность + путь/глоб.
  - `{ surface: "claude-global", path: "CLAUDE.md" }`
  - `{ surface: "claude-global", path: "commands/**" }`
  - `{ surface: "project", project: "erp", path: ".claude/**" }`
  - `{ surface: "folder", anchor: "epic-payments", path: "context/**" }` — произвольная папка
- **Capability** (схема сейчас, исполнитель позже): `{ kind: "plugins", ids: [...] }`, `{ kind: "mcp", servers: {...} }`.

Записи группируются в **категории** для one-click («глобальные правила», «commands», «skills», «settings», «project: erp», «team: epic-payments», «plugins»…). Это и умные дефолты, и произвольные пути одновременно.

**Слои:**
- Манифест в репе = *offered set*.
- Локальный конфиг устройства = что активно здесь (вкл/выкл по записи/категории) + device-local добавки.
- Дефолт при первом подключении источника: всё offered активно («всё ON»), дальше подрезаешь.

**Мульти-источник:** N источников (личная + командные), git-истории не смешиваются. Эффективный синк = объединение активных записей. Коллизия целевых путей из двух источников — показывается, не сливается молча.

**Якоря произвольных папок (cross-machine):** манифест хранит логическое имя-якорь, устройство мапит его в локальный путь (та же нормализация, что для `claudeProjects` name↔path). `epic-payments` → A `/Users/a/work/epic`, B `C:\work\epic`.

**EPIC-сценарий:** лид добавляет в командную репу запись на папку эпика + категорию `team: epic-payments`, пушит. Участник подключает источник → видит категорию (активна по умолчанию) → синкает только её, остальное может оставить выключенным.

### 4.3 Операции и UX (1–3 клика)

Сохраняем из текущего движка: preview, 3-way resolver, canonicalization settings, нормализацию имён, git-plumbing через temp-index, scoped `git clean`. Меняется *что считается изменением* и *как показывается*.

- **Статус** (фоном, без мутаций диска): по источнику × активным записям diff «живое ↔ репа ↔ remote». Чип: входящие / исходящие / конфликты / **warnings** (tracked не читается; tracked пропал — кандидат на удаление).
- **Push:** коммит из активных tracked-записей. Не прочитался → версия в репе сохраняется + warning. Пропал → кандидат на удаление (не вырезается сам). Floor-проверка. Preview раздельно: `добавлено / изменено / удаляется (подтвердить) / пропущено (warning)`. ← фикс F1-surfacing.
- **Pull:** apply remote→живое, атомарно+аддитивно. Удаления opt-in, settings-merge, diverged блокирует (всё уже есть).
- **One-click «Применить источник»:** свежий участник подключил источник → один клик → всё offered&активное приехало (позже + capability-apply). Повседневно: чип показывает дрейф → кнопка **Sync** = безопасный push+pull с preview.
- **«Тонкий конфиг» = UI:** по источнику список предложенных категорий/записей с тогглами «активно на этом устройстве». Обобщение toggle-UI из Settings → Claude.
- **Capability-записи:** показываются как категории, но исполнитель отложен (несутся в манифесте без применения, помечены «скоро»).

### 4.4 Миграция — чисто аддитивная

1. **Синтез манифеста из текущего конфига** при апгрейде (CLAUDE.md/commands/skills/settings по `syncGlobal`; per-project memory/.claude по флагам; cursor-проекты) → пишется `.claudesync/manifest.json`. Существующий `claude/`/`cursor/` контент не трогается.
2. **Конфиг мапится 1:1:** нынешние тогглы → «активные записи» устройства.
3. **Раскладка репы та же** (`claude/...`, `cursor/...`) → старые версии частично работают, без деструктивной реструктуризации.
4. **Первый sync после миграции** под новыми инвариантами: манифест совпадает с репой → удалять нечего; floor + explicit-deletion подстрахуют.

Ни одной деструктивной операции в самой миграции.

## 5. Порядок сборки (4 независимо релизимых под-проекта)

**№1 — Предохранители на ТЕКУЩЕЙ модели (СРОЧНО, останавливает потерю данных).**
Без манифеста. Содержание:
- «ошибка чтения ≠ удаление» (F1): не прочитался tracked-файл → сохранить версию в репе + warning.
- Floor на массовые удаления (F1b): отказ + подтверждение, если удаляется большая доля / источник пуст при непустом HEAD.
- Явное подтверждение удалений в push-preview + раздельная секция «удаляется» (F1-surfacing): `preview-push-status` должен отдавать `status`, PushModal — отдельную секцию удалений с чекбоксами.
- Оживить settings.json JSON-guard (F1): битый JSON → прервать push с понятной ошибкой, не эмитить deletion.
- Атомарная запись на apply.
- (Доп.) F4 — явное предупреждение в Discard про удаление локально-новых файлов.
Подмножество целевой модели, не выкидной код. Маленький объём, лечит боль немедленно.

**№2 — Манифест как авторитет членства.** Манифест (синтез при миграции) вместо хардкод-правил. Слои репа/локаль. Обобщение toggle-UI. F2/F3 закрываются здесь (reverse-mirror и install идут от манифеста единообразно).

**№3 — Мульти-источник.** N реп, личная + командные, якоря произвольных папок, EPIC-сценарий.

**№4 — Capability-исполнитель.** Плагины/MCP desired-state apply.

Каждый под-проект самостоятельно тестируется (round-trip харнес уже есть: `tests/fixtures/sync-roundtrip.ts`, `tests/main/engine/sync-roundtrip.test.ts`). №1 — ценность немедленно; №2–3 — продуктовое видение; №4 — командный capability-слой.
